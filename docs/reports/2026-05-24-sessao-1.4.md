## Decisões tomadas

- **D1** — Adapter Prisma excluído do coverage unitário: requer DB real, alinhado com estratégia da sessão 1.5 (Testcontainers). Padrão consistente com `outcome.ts` e `server.ts`.
- **D2** — Router testado com Node fetch nativo (sem supertest): `supertest` não estava instalado e o registry npm estava bloqueado no sandbox. Servidor HTTP real em porta aleatória, teardown garantido via `afterEach`.
- **D3** — Branches defensivos removidos de `pauseAgent`/`activateAgent`: os 4 status do AgentStatus são exaustivos, `AgentInvalidStatusTransitionError` tornaria código morto. Mantido no `agent.errors.ts` para uso futuro (ex: sessão 1.5 pode adicionar status).
- **D4** — cross-tenant sempre retorna 404: critério #2 do E1 — não vazar existência de agentes de outros tenants.
