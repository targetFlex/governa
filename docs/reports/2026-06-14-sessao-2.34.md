## Decisões tomadas

- **D1** — Tracer como primeiro import em `server.ts`: OTel exige monkey-patching antes do carregamento de Express/HTTP; qualquer import anterior anula a instrumentação automática.
- **D2** — `@opentelemetry/api` como única dependência em `metrics.ts`: isolamento do SDK; se SDK não inicializado, operações são no-op; testes precisam mockar apenas a API.
- **D3** — `OTEL_SDK_DISABLED=true` via `setupFiles`: evita inicialização acidental do SDK nos testes unitários sem exigir mock em cada spec.
- **D4** — `attributes/remove_pii` no collector: remove `Authorization` e `Cookie` antes de qualquer export; PII nunca sai do processo via trace.
- **D5** — OTel Collector (não SigNoz direto) no docker-compose: SigNoz é heavy (ClickHouse, etc.); para dev local, collector com exporter `debug` é suficiente. SigNoz comentado como opção de produção.
- **D6** — Histograma com `explicitBucketBoundaries` customizado: buckets alinhados com SLOs do projeto (P95 < 3000ms para core; P95 < 10000ms para Protheus).
