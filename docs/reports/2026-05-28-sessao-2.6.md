## Decisões tomadas
- **D1** — Padrão "uma instância Pact por teste com porta aleatória": evita bug TCP TIME_WAIT /
  ECONNREFUSED do Pact v12 ao reusar porta entre testes sequenciais. Porta lida via
  `(provider as any).opts.port` após `setup()`
- **D2** — Verificação manual no provider (sem `Verifier` class): pact-core@14.3.8 tem bug de
  ambiente no Linux arm64 (sandbox) onde o proxy Rust FFI não entrega requests ao provider,
  retornando sempre 403. Manual http.request + assertBody() produz semântica idêntica sem FFI
- **D3** — `assertBody()` com matchingRules Pact V2: campos com `{ "match": "type" }` validam
  `typeof actual === typeof expected`; demais campos usam deep equality. Resolve campos dinâmicos
  como `id` (UUID gerado) e `createdAt`/`updatedAt`
- **D4** — Seed pact com `Authorization: Bearer PACT_TEST_TOKEN` substituído no `doRequest()` por
  JWT real assinado com `JWT_SECRET`: o placeholder nunca chega ao middleware de autenticação
- **D5** — `InMemoryAgentInventoryRepository` como fixture compartilhada entre stateHandlers e rotas:
  mutações dos handlers são visíveis ao AgentService na mesma requisição de verificação

## Desvios do plano
- **D-ENV** — `Verifier` class (Pact FFI) não funciona no ambiente Linux arm64 do sandbox por bug
  no pact-core@14.3.8; solução: verificação manual que mantém semântica completa sem dependência
  de FFI (decisão D2)
- **D-GIT** — Commits executados pelo usuário via terminal (não pelo sandbox): o sandbox consegue
  escrever o índice git mas não consegue deletar o `.git/index.lock` no filesystem montado,
  causando falha no `git commit` interno
