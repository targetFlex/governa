- **D1** — `userId` decodificado no frontend via `atob(token.split('.')[1])` sem biblioteca adicional — JWT é apenas lido (não verificado), o que é aceitável para extrair claims já validados pelo backend
- **D2** — `createdAgent: Agente | null` como signal de sucesso — o componente usa `effect()` para navegar quando não-null, padrão análogo ao `saveSuccess` do politica-form
- **D3** — Rota `/agentes/novo` posicionada antes de `/agentes/:id` — Angular Router é first-match; colocar depois causaria a string "novo" ser interpretada como `:id`
- **D4** — Lista de modelos e ferramentas hardcoded no componente — suficiente para MVP; externalizar para endpoint `/config` é pós-MVP
