## Decisões tomadas
- **D1** — `instance.request(config)` em vez de `instance(config)` no retry: mais fácil de mockar em testes unitários sem `axios-mock-adapter`
- **D2** — Estratégia de teste para `http-client`: capturar interceptors via `axios.create()` mockado e exercitá-los diretamente — elimina dependência de `axios-mock-adapter` não instalável no sandbox
- **D3** — `apps/governa-gateway/` criado dentro do monorepo `governa` para aproveitar `node_modules` do workspace; repositório standalone mantido sincronizado para futuro `git remote add origin`
- **D4** — `jest.config.js` (não `.ts`) para compatibilidade com o jest do governa-core sem transpile adicional

## Desvios do plano
- `governa-gateway` criado como repositório standalone **e** como `apps/governa-gateway` no monorepo (não apenas standalone como planejado): o sandbox não conseguiu instalar `axios` via npm; a solução foi usar o `node_modules` do `governa-core` via symlink, o que exigiu situar o gateway dentro do monorepo para os testes. O código é idêntico nos dois locais.
- `axios-mock-adapter` não foi usado nos testes de `http-client`: substituído por captura direta dos interceptors (mesma cobertura funcional, abordagem mais isolada).
