## Decisões tomadas
- **D1** — Arquivos criados manualmente (sem `ng new`): sandbox sem acesso ao npm registry; estrutura 100% equivalente ao output do Angular CLI
- **D2** — Token JWT em memória (signal), não em localStorage/sessionStorage: padrão de segurança para mitigar XSS
- **D3** — Functional guard + functional interceptor (Angular 15+ style): alinhado com standalone components e tree-shaking
- **D4** — `@env/*` como path alias: evita imports relativos longos e facilita override por ambiente
- **D5** — NGRX Signals para estado de auth: reativo, sem boilerplate de Actions/Reducers para estado simples de sessão
