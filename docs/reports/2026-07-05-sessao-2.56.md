## Decisões tomadas

- **D1** — `AgenteDetailComponent` usa `HttpClient` direto (sem `AgentesStore`) — o store gerencia a lista; o detalhe é contexto próprio sem estado compartilhado relevante
- **D2** — Erro 404 exibe estado "Agente não encontrado" separado do banner de erro genérico — intenção de UX diferente (não é falha técnica)
- **D3** — Fallback de erro usa `err.error.message ?? 'mensagem padrão'` (sem `err.message`) — Angular preenche `err.message` com a URL técnica da requisição, que não é mensagem útil para o usuário
- **D4** — Links de `agentId` no dashboard mantêm estilo `cockpit__td--mono` (herança de cor) e só mudam cor ao hover — não devem distrair visualmente nas tabelas de auditoria/alertas
