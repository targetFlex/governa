## Decisões tomadas
- **D1** — Spec de barrel em vez de mudar import no auth client: evita alterar código de produção
  para forçar cobertura. O spec documenta explicitamente que handleAuthError é re-exportado e
  funcional via este barrel.
- **D2** — Mock Protheus com `http.createServer` in-process (sem express no governa-gateway): gateway
  é uma biblioteca de conectores, não um servidor — adicionar express como dependência seria escopo
  errado. Mock Node.js nativo é suficiente para testar HTTP real.
- **D3** — Timer tracking no mock server: `req.on('close')` + `server.on('close')` cancela timers
  pendentes do E2E-7 (timeout test), eliminando o aviso "Jest did not exit" sem precisar de
  `--forceExit`.
- **D4** — `jest.setTimeout(15_000)` na suite E2E mas timeout individual de 10s no E2E-7: cada
  teste tem orçamento explícito, evitando falsos negativos por timeout de suite.
- **D5** — PiiPseudonymizer com chave de teste fixa em E2E: permite verificar que documentoPseudo
  tem 64 chars (HMAC SHA-256) e não contém o CNPJ real, sem depender de env var.

## Desvios do plano
- **D-SCOPE** — Não foi criado um servidor Express para governa-gateway (gateway HTTP server): o
  handoff mencionava "servidor Express in-process", mas governa-gateway é uma biblioteca de
  conectores sem express como dependência. A alternativa de mock Protheus in-process testa o
  mesmo fluxo de forma mais coesa com a arquitetura atual. Registrar para avaliar se um gateway
  HTTP server é necessário em sessões futuras (para satisfazer os contratos CC1-CC4 do governa-core).
