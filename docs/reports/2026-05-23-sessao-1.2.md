- **D1** — Hexagonal real em vez de PrismaClient direto no PolicyEngine · endurece critério #6 ("nenhuma query Prisma sem tenantId") por construção, não por revisão
- **D2** — Custom error classes (`AgentNotFoundError`, `AgentWithoutPolicyError`) · retrocompat com `throw new Error` + `instanceof` em handlers HTTP
- **D3** — Tool registry com 5 reads + 2 writes (não só os 3 do seed) · necessário para exercitar testes de prefixo/exato em ASSISTIDO/AUTONOMO
- **D4** — Gherkin-style nos describes sem Cucumber · atende preferência #3 em forma sem custo de dependência
- **D5** — `createTenantMiddleware({secret})` factory além do default · testabilidade sem mutar `process.env`
- **D6** — Migration da role promovida com SQL idempotente (`DO $$ IF NOT EXISTS ... $$`) · permite re-aplicação segura em CI ephemeral
- **D7** — Protocolo de reports incremental (handoff por arquivo curto datado) · resolve consumo alto de tokens no boot — cada sessão lê só o último report, não SOLUTION/E1 inteiros
- **D8** — Nomenclatura `YYYY-MM-DD-sessao-X.Y.md` em `docs/reports/` · sort lexicográfico = cronológico, permite múltiplas sessões/dia
- **D9** — Skill versionado em `skills/` (não em `_skill-staging/`) · fonte canônica no git, install via `cp skills/... ~/.claude/skills/`
