## Decisões tomadas
- **D1** — `server.ts` separado de `gateway-app.ts`: mantém hexagonal — `gateway-app.ts` é porta pura (testável sem env), `server.ts` é a raiz de composição
- **D2** — `env_file` não usado no docker-compose; variáveis com `${VAR:-default}`: permite override via `.env` do host sem exigir arquivo extra no container
- **D3** — `/health` adicionado antes das rotas autenticadas: healthcheck do Docker não depende de credenciais Protheus; resposta mínima `{ status: ok }`
- **D4** — `tsx` adicionado como script `dev` mas NÃO como dep nova: `tsx` já está em `devDependencies` do governa-core (pnpm workspace compartilha); gateway usa em dev apenas
