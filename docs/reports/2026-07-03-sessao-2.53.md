## O que mudou no código

- `infra/docker/docker-compose.prod.yml` — stack de produção: Traefik + postgres + redis + otel-collector + governa-gateway + governa-core; sem adminer, sem pact-broker; redes `governa_public` (pública, só Traefik e core) e `governa_backend` (internal: true)
- `infra/docker/traefik-tls.yml` — dynamic config Traefik: `minVersion: VersionTLS13`, `sniStrict: true`, cipher suites explícitos
- `infra/docker/.env.production.example` — template completo de variáveis de produção com comentários de geração de segredos
- `scripts/deploy.sh` — pull imagens GHCR → `up -d --remove-orphans` → health check com timeout 120s e log de falha
- `.github/workflows/deploy.yml` — job `build-push` (matrix governa-core/governa-gateway → GHCR) + job `deploy` (SSH + `appleboy/ssh-action@v1.2.0`); trigger em push para `main` em paths relevantes + `workflow_dispatch`

## Decisões tomadas

- **D1** — Traefik v3.1 (não v2): v3 é a versão atual estável; TLS options via arquivo de config dinâmico (`traefik-tls.yml`) montado em `/etc/traefik/dynamic/`
- **D2** — ACME staging como default: `ACME_CA_SERVER` no `.env` controla o issuer; muda para prod URL sem alterar código quando DNS estiver propagado
- **D3** — Rede backend `internal: true`: governa-gateway, postgres, redis e otel-collector não têm binding de porta; só Traefik e governa-core ficam na rede pública
- **D4** — `IMAGE_OWNER` como variável no compose: permite mudar de username pessoal para org sem alterar o arquivo compose
- **D5** — `GHCR_TOKEN` separado do `GITHUB_TOKEN`: o VPS precisa de autenticação persistente no GHCR; um PAT com `read:packages` é mais seguro que passar o `GITHUB_TOKEN` por SSH env
- **D6** — Deploy job com `environment: production`: habilita proteção via GitHub Environments (revisores obrigatórios podem ser adicionados depois sem mudar o workflow)

## Secrets necessários no repositório GitHub

| Secret | Descrição |
|---|---|
| `VPS_HOST` | IP ou hostname do VPS |
| `VPS_USER` | Usuário SSH no VPS |
| `VPS_SSH_KEY` | Chave SSH privada (Ed25519) |
| `GHCR_TOKEN` | GitHub PAT com escopo `read:packages` |

## Setup do VPS (one-time, manual)

```bash
# No VPS, como root ou usuário com sudo:
apt-get install -y docker.io docker-compose-plugin git
systemctl enable --now docker

git clone git@github.com:targetFlex/governa.git /opt/governa
cp /opt/governa/infra/docker/.env.production.example /opt/governa/infra/docker/.env
# Editar .env com valores reais

# Abrir portas no firewall
ufw allow 22 && ufw allow 80 && ufw allow 443 && ufw enable
```
