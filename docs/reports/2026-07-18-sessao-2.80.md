# Report — Sessão 2.80 · Mismatch de contrato Cliente/Pedido (governa-ui ↔ governa-core)

**Data:** 2026-07-18
**Status:** ✅ concluída (fix em produção validado end-to-end via API)
**Sessão anterior:** [`2026-07-17-sessao-2.79.md`](./2026-07-16-sessao-2.79.md)

## TL;DR

Item registrado como "achado não tratado" no report 2.79: os modelos de frontend
(`cliente.model.ts`/`pedido.model.ts`) esperavam campos em texto claro (`nome`,
`documento`, `clienteNome`) que não existem no `ClienteInterno`/`PedidoInterno` do core
desde o D16 (PII pseudonimizada via HMAC). Investigação confirmou não só o mismatch,
mas um **bug de produção ativo**: `pedido-card.component.ts` indexava `STATUS_META`
só com o enum antigo do frontend (`ABERTO`/`EM_APROVACAO`/`APROVADO`/`CANCELADO`/`ENCERRADO`)
— qualquer pedido com status real `BLOQUEADO` ou `LIBERADO` (que **já está** nos dados
sintéticos de produção, pedidos `10003`/`10004`) fazia `statusMeta` vir `undefined` e o
template quebrava (`Cannot read properties of undefined`). Usuário escolheu resolver via
endpoint de reidentificação (PII sob demanda, auditado) em vez de expor os tokens crus.

## Achados confirmados na investigação

- **Cliente:** frontend esperava `id, codigo, nome, tipoPessoa, documento, email,
  telefone, ativo, limiteCredito, saldoDevedor, moeda, criadoEm, atualizadoEm`. Backend
  (`ClienteInterno`) só tem `clienteId, loja, nomeToken, documentoToken, enderecoToken,
  emailToken, telefoneToken, bloqueado`. Zero sobreposição de nomes — routers do core
  serializam a entidade crua sem mapper (`res.json({ data: output.clientes })`).
- **Pedido:** frontend esperava `numero, clienteNome, valor, moeda, dataEntregaPrevista,
  itens[{codigo,descricao,valorUnitario,valorTotal}]`. Backend (`PedidoInterno`) tem
  `numeroPedido, clienteId, loja, valorTotal, status (ABERTO|BLOQUEADO|ENCERRADO|LIBERADO),
  itens[{codigoProduto,quantidade,precoUnitario}]` — nunca teve nome de cliente.
- **Crash confirmado (não só campo em branco):** `STATUS_META[pedido.status]` retorna
  `undefined` para `BLOQUEADO`/`LIBERADO` → template acessa `.bg`/`.color`/`.label` de
  `undefined` → exceção no render do card. Testes existentes (`*.spec.ts`) mockavam o
  shape antigo com status fictícios (`EM_APROVACAO`) e nunca capturaram a divergência
  contra o shape real.
- **Track key quebrado:** `@for (cliente of store.clientes(); track cliente.id)` — `id`
  nunca existiu no payload real, sempre `undefined`, perdendo garantia de identidade
  única na lista.

## Decisão de arquitetura (usuário)

Três opções levantadas: (a) endpoint de reidentificação (governa-core resolve PII sob
demanda via novo endpoint no gateway, auditado); (b) expor só os tokens, sem PII legível;
(c) fix mínimo do crash sem resolver reidentificação. **Usuário escolheu (a)**.

## O que mudou no código

### `governa-gateway`
- `cliente.schema.ts`: novo tipo `ClientePiiView` (PII em texto claro — uso restrito).
- `cliente.mapper.ts`: `toPlaintextView(raw)` — mapeia `ProtheusCilenteRaw` para PII em
  claro, sem pseudonimizar (contraparte de `toInterno`).
- `read-protheus-cliente.connector.ts`: `executePii({codigoCliente, loja})` — busca
  registro único, retorna `null` se não encontrado (nunca lança NotFound).
- `synthetic-cliente.connector.ts` + `synthetic-fixtures.ts`: `executePii()` +
  `SYNTHETIC_CLIENTES_PII` (nomes claramente fictícios) para o piloto sandbox.
- `gateway-app.ts`: nova rota `GET /clientes/pii?codigoCliente=X&loja=Y` (400 se faltar
  parâmetro, 404 se não encontrado).

### `governa-core`
- `gateway-client.port.ts`: `ClientePiiView`, `ReidentificarClienteParams`,
  `IGatewayClient.reidentificarCliente()`.
- `http-gateway-client.ts`: implementação + `getSingle<T>()` (helper novo — 404 vira
  `null`, distinto do `get<T>()` de listas que vira `[]`).
- `reidentificar-cliente.use-case.ts` (novo): `ReidentificarClienteUseCase` — grava
  `AuditEvent` **sempre** (sucesso ou falha), `purpose: 'exibicao_painel_humano'`,
  `toolCalled: 'read_protheus_cliente_pii'`, `dataCategories: ['identificacao',
  'dados_pessoais']` — distingue no audit trail o momento em que um humano viu PII em
  claro do momento em que um agente consultou dados pseudonimizados.
- `panel-access.resolver.ts`: `resolvePanelSubjectAccess()` — mesmo padrão de
  `resolvePanelAccess` (2.78), mas `subjectToken` específico por
  `(tenant, clienteId, loja)` em vez de listagem genérica. Não depende de `PII_HMAC_KEY`
  (confirmado ausente em produção desde 2.78).
- `clientes.router.ts`: nova rota `GET /:clienteId/reidentificar?loja=X` — **exclusiva
  do painel humano**, sempre resolve via `resolvePanelSubjectAccess`, nunca aceita
  `agentId`/`subjectToken` de query string (diferente de `GET /`, que aceita ambos os
  caminhos).
- `app.ts`/`server.ts`: wiring de `reidentificarClienteUseCase`.

### `governa-ui`
- `cliente.model.ts`/`pedido.model.ts`: reescritos para o shape real do core +
  `ClientePii`/`ClientePiiResponse`.
- `cliente-pii.service.ts` (novo): `ClientePiiService.reveal(clienteId, loja)` — chama
  `GET {coreBaseUrl}/clientes/:id/reidentificar`, cacheia por sessão (evita reconsultas
  redundantes — cada reidentificação gera `AuditEvent`), evicta cache em erro (permite
  retry).
- `cliente-card.component.ts`: PII oculta por padrão (mostra `clienteId`/`loja`), botão
  "Revelar dados" chama `ClientePiiService` sob demanda — nunca em bulk na listagem.
- `pedido-card.component.ts`: `STATUS_META` corrigido para os 4 status reais
  (`ABERTO`/`LIBERADO`/`BLOQUEADO`/`ENCERRADO`); cliente também revelado sob demanda
  (reusa `ClientePiiService` com `clienteId`+`loja` do próprio pedido).
- `clientes-list.component.ts`/`pedidos-list.component.ts`: `track cliente.id` →
  `track cliente.clienteId`; `track pedido.id` → `track pedido.numeroPedido`.
- `clientes.service.ts`: `clientesAtivos` computed usa `!c.bloqueado` (não existe mais
  `ativo`).
- Specs e stories de todos os arquivos acima atualizados para o shape real; spec novo
  `cliente-pii.service.spec.ts` (5 casos: happy path, cache por chave composta, chamadas
  independentes por cliente distinto, eviction em erro + retry, `clear()`).
