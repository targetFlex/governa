### Opção A — Merge do branch + limpeza

- Merge de `feat/session-2.45-notification-config` → `main`

**Risco:** baixo — o branch está estável, é merge direto.

**Esforço estimado:** 30–60 min.

### Opção B — E5 backend: OTel + SigNoz no governa-core

**O que envolve** (sessão 5.1 do `GOVERNA_AGENTES_MVP_E5.md` — *doc ainda não criado*):
- Instrumentação OpenTelemetry em todos os serviços do governa-core (HTTP, DB, Redis/BullMQ)
- Exportador OTel → SigNoz (self-hosted ou cloud trial)
- Traces de ponta a ponta: request Angular → governa-api → worker → Postgres
- Métricas: error rate, latência P95, volume de decisões de agente

**Quando faz sentido:** E5 UI (2.49) está pronta; a stack de observabilidade é o próximo bloco lógico do E5 completo.

**Pré-requisito técnico:** doc `GOVERNA_AGENTES_MVP_E5.md` precisa ser criado (ou o escopo da sessão precisa ser definido inline).

**Risco:** médio — requer decisão sobre SigNoz local vs cloud, OTel SDK version, e quais serviços do core instrumentar primeiro.

**Esforço estimado:** 2–3 sessões.

---

### Opção C — Testes E2E Playwright em `/alertas`

**O que envolve:**
- Setup inicial do Playwright no governa-ui (se ainda não existir)
- Testes E2E: carregamento da lista de alertas, abertura/fechamento do painel de notificações, preenchimento e salvamento de configuração email + webhook

**Quando faz sentido:** o `NotificationConfigPanelComponent` nunca foi testado ponta a ponta; os 413 testes são unitários/integração; antes de mergear para main é prudente ter pelo menos um E2E smoke test.

**Risco:** baixo-médio — Playwright precisa de backend real ou mock configurado.

**Esforço estimado:** 1–2 sessões.

## Análise comparativa

| Critério | A — Merge | B — OTel | C — E2E |
|---|---|---|---|
| Valor imediato | Limpa dívida técnica de branch | Fecha o E5 completo | Cobre gap de qualidade |
| Risco | Baixo | Médio | Baixo-médio |
| Esforço | ~1h | 2–3 sessões | 1–2 sessões |
| Bloqueia algo? | Não | Não | Não |
| Dependência nova | Nenhuma | SigNoz + OTel SDK | Playwright |
| Pode ser paralelo? | Pré-requisito natural | Independente de C | Independente de B |

**Combinação recomendada sugerida:** A → C → B
- Merge limpa o branch e fecha o ciclo E5 UI
- E2E cobre o gap de qualidade antes de avançar no backend
- OTel/SigNoz vira E5 backend com escopo próprio

Mas o usuário decide — qualquer ordem é válida.
