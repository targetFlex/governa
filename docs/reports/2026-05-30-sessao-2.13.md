## O que foi feito
- Criado `src/connectors/auth/auth-login.connector.ts`:
  - Interface `IAuthLoginConnector` com `execute(params: LoginParams): Promise<LoginResult>`
  - Classe `ProtheusLoginConnector` — Resource Owner Password Grant via Protheus `POST /oauth2/token`
  - Erros mapeados via `handleUpstreamError` (401→PROTHEUS_UNAUTHORIZED, 5xx→PROTHEUS_INTERNAL_ERROR, timeout→PROTHEUS_TIMEOUT)
- Criado `src/connectors/auth/auth-login.connector.spec.ts` — 6 casos (AL-1 a AL-6)
- Atualizado `src/gateway-app.ts`:
  - Import + re-export de `IAuthLoginConnector`
  - `authLoginConnector` como 3º arg do construtor (required)
  - Rota `POST /auth/login` com validação de `email` e `password`
  - Handler `sendLoginError` — 401 para PROTHEUS_UNAUTHORIZED, 502 para demais upstream
- Atualizado `src/gateway-app.spec.ts` — 7 novos casos (AL-OK, AL-400×3, AL-401, AL-502, AL-500) + 3º arg em todos os construtores
- Atualizado `test/pact/provider/gateway.provider.pact.spec.ts` — 3º arg no construtor

## Decisões tomadas
- **D1** — `authLoginConnector` required (não optional): força contrato explícito, evita silêncio em produção
- **D2** — Resource Owner Password Grant: alinhado com o `ProtheusAuthClient` existente e testável sem Protheus real
- **D3** — `sendLoginError` separado de `sendError`: respostas 401 são específicas de auth; não polui o handler genérico
