## Decisões tomadas

- **D1** — `coreBaseUrl` separado de `gatewayBaseUrl`: `/agents` só existe no core (:3000); misturar as URLs criaria acoplamento incorreto.
- **D2** — `acoesEmAndamento: string[]` no state (por id): permite spinner por card sem bloquear a lista inteira — UX superior a um flag global.
- **D3** — Filtro por status no frontend (não nova chamada HTTP): inventário de agentes é tipicamente pequeno; evita round-trip desnecessário.
- **D4** — Botão Ativar `disabled` quando `policyId` é null (espelha regra `AgentNoPolicyError` do backend): falha rápida no cliente, sem chamada HTTP destinada a falhar.
- **D5** — Mirror fiel do padrão `ClientesList`/`PedidosList` para skeletons, error banner e empty state: consistência de UX.
