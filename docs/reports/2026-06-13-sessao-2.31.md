## Decisões tomadas

- **D1** — PDF via `window.print()` + template HTML dedicado: npm install quebrado na sandbox (`Cannot read properties of null (reading 'matches')` no arborist v10.9.8); abordagem nativa é zero-dep, funciona universalmente, suficiente para regulador (ANPD).
- **D2** — `GET /audit-events/export` antes de `GET /` no router: rota `/export` deve ser registrada antes de qualquer parâmetro dinâmico para não ser capturada como `:id`.
- **D3** — `expect.any(Array)` dentro de `toMatchObject` falha silenciosamente no Jest 29.7.0 neste ambiente; substituído por `Array.isArray()` — comportamento verificado e documentado no spec.
- **D4** — `server.ts` corrigido para incluir `PolicyService` e `AuditQueryService` (eram omitidos da composição raiz desde a sessão 2.30 — não afetava testes mas impedia start real).
- **D5** — `listForExport` limitado a 10 000 registros internamente: protege memória sem expor parâmetro de limite ao caller.
