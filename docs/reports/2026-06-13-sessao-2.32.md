## Decisões tomadas

- **D1** — SSE via `EventSource` nativo (zero dependência): WebSocket exigiria lib extra (npm install quebrado na sandbox). `EventSource` é suportado nativamente em todos os browsers modernos e no Express sem dependência adicional.
- **D2** — `GET /alerts/stream` registrado ANTES de `/thresholds` e `/:id` no router: rota literal `/stream` deve preceder parâmetros dinâmicos.
- **D3** — `InMemoryAlertRepository` em `server.ts` até migration Prisma: tabelas `alerts` e `alert_thresholds` não existem no schema ainda — instância in-memory garante que o servidor inicia e a UI funciona end-to-end.
- **D4** — `beforeEach(() => process.env.JWT_SECRET = 'test-secret')` obrigatório no router.spec: `tenantMiddleware` retorna 500 (não 401) quando `JWT_SECRET` não está definido — todos os testes do router falhavam com 500 por essa razão.
- **D5** — `[...ALERT_KINDS] as unknown as [AlertKind, ...AlertKind[]]`: arrays `as const` são readonly; o cast direto `as [...]` falha no TypeScript strict — spread cria cópia mutável antes do cast.
- **D6** — `EventSource` usa query param `?token=` para JWT: a API nativa não suporta headers customizados; convenção segura para SSE em ambiente controlado.
