## Decisões tomadas
- **D1** — Verificação manual (sem Pact Verifier binário): o Pact Verifier nativo retornava 403 em todas as rotas (bug Rust FFI / estado Linux arm64 já documentado em governa-core). Seguido o mesmo padrão de `core.provider.pact.spec.ts`: ler pact JSON + stateHandlers in-process + HTTP requests manuais.
- **D2** — `IPedidoConnector` / `IClienteConnector` interfaces como portas de entrada: permite injeção de mocks nos testes sem precisar instanciar connectors reais (Hexagonal Architecture). Connectors concretos (`ReadProtheusPedidoConnector`, `ReadProtheusCilenteConnector`) continuam intactos.
- **D3** — `close()` idempotente via `ERR_SERVER_NOT_RUNNING`: o ciclo `beforeEach/afterEach` dos testes unitários fecha o servidor duas vezes (teste + afterEach). Tratar como sucesso elimina o falso negativo.
- **D4** — Express instalado via symlink manual no sandbox (pnpm não disponível, npm bloqueado por política). Package.json declarado com versão correta para que `pnpm install` no host resolva normalmente.
